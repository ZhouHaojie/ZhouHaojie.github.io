Dynamische Variablendeklarationen: Zeiger vs Felder

C/C++ erlaubt einen flexiblen Umgang mit dem Speicher während der Ausführung des Programms. Dazu werden Zeiger verwendet, welche einen direkten Zugriff (访问) auf dem Speicher ermöglichen.

Vorteil

  • Flexibilität: erst zu Laufzeit wird die Dimensionierung von dynamischen Feldern festgelegt 动态内存分配
  • Komplexe Datenstrukturen können leichter verwirklicht werden: Listen, Bäume… 用于表示和实现各种复杂的数据结构

Nachteil

  • Fehlersuche erschwert
  • Laufzeitnachteile

Objekte: Eine Variable, Konstante in einem Programm ist ein Objekt. Jedes Objekte hat einen gewissen Speicherbedarf.

Zeiger und Aderessen: const

Felder(数组)

Speicherdarstellung

  • Eindimensionales Felder

    tab[i] ist äquivalent zu *(tab+i), wobei von Ausgangsspeicherstelle tab um i Positionen erhöht wird.

    tab Datentyp: Zeiger auf int

  • Mehrdimensionale Felder

    mat[i][j] ist äquivalent zu *(mat[i]+j) = *(*(mat+i)+j)

    mat[i] ist Zeiger auf i-te Zeile

    mat Datentyp: Zeiger auf Zeiger von Typ int

Größe eines Objekte: sizeof()

std::size_t sizeof (Argument) gibt ein Konstante vom Datentyp std::size_t zurück: Anzahl der Bytes

Rechner: Speicherorganisation

automatisch Objekte werden auf dem Stack (Stapel) angelegt ; globale Objekte (Nutzer verwaltet) werden auf dem Heap angelegt.

  • Stack:
    • Last In First Out
    • Lokale Variablen in Funktionen werden automatisch verwaltet: angelegt und gelöst
    • Variablen auf dem Stack existieren nur solange man in der aufrufenden Funktion ist(Ausnahme Variablen mit static Attribut)
    • Stack: hat eine Größenbeschränkung (die man einstellen kann)
  • Heap:
    • Region im Speicher des Rechners, in welchem nicht automatisch angelegte Objekte hinterlegt werden, also sobald Sie in C++ new und delete Operatoren verwenden
    • keine Größenbeschränkung

Dynamische Variablendeklaration: new, delete

  • Felder, die mit new [] angelegt(创建) wurden, müssen mit dem delete [] Operator gelöscht werden, da sonst nur das erste Element des Feldes gelöscht wird und der Rest den Speicher weiterhin belegt, ohne dass darauf zugegriffen werden kann!!! memory leak 否则只有该字段的第一个元素将被删除，其余的内存仍将被使用而无法访问！ 内存泄漏
  • Nach Anwendung von delete ist Zeiger undefiniert! (dangling pointer) 使用delete后，指针未定义！ （悬空指针）

Dynamische Feld 2d

  1. Variante für zweidimensionales Feld mit nicht zusammenhängender Speicherstruktur

  2. zweidimensionales Feld mit zusammenhängendem Speicher(连续内存空间)

    Im nebenstehenden Quellcode wurde darauf geachtet, dass der Speicherbereich des Feldes zusammenhängend ist: dies kann Geschwindigkeitsvorteile bringen.

Funktionszeiger(函数指针)

  • Argument: ein Objekt vom Datentyp float
  • Rückgabe: ein Objekte vom Datentyp float

const und constexpr

  • const: wird nachfolgend nicht verändert

  • constexpr: wird zur Zeit des Übersetzens ausgerechnet

    对于修饰Object来说，const并未区分出编译期常量和运行期常量（指变量不能被修改），constexpr限定在了编译期常量，在编译期可以被计算出来。

    • Function muss constexpr Attribut haben constexpr修饰的函数，简单的来说，如果其传入的参数可以在编译时期计算出来，那么这个函数就会产生编译时期的值。但是，传入的参数如果不能在编译时期计算出来，那么constexpr修饰的函数就和普通函数一样了。不过，我们不必因此而写两个版本，所以如果函数体适用于constexpr函数的条件，可以尽量加上constexpr。而检测constexpr函数是否产生编译时期值的方法很简单，就是利用std::array需要编译期常值才能编译通过的小技巧。这样的话，即可检测你所写的函数是否真的产生编译期常值。

    c++11-Fortschrittliches Speichermanagment

    C++中采用new和delete来创建（申请）和释放（销毁）内存，但是因为释放内存时机把握不当、忘记释放内存等原因，很难准确及时地管理内存。
    因此，C++标准库推出了智能指针机制来更好的管理内存，智能指针能自动释放所指向的对象。

    include<memory>

    Neue Praktische Zeigertypen in Bibliothek \(智能指针)

    • shared_ptr : verwaltet Zeiger; Möglichkeit das Objekt mit anderen shared_ptr zu teilen

      允许多个指针指向同一个对象，当进行拷贝或者赋值操作时，每个shared_ptr都有一个关联的计数器，通常称其为引用计数。

      我们每次拷贝一个shared_ptr都会使这个引用计数递增（当局部的shared_ptr离开作用域时，计数器就会递减），当引用计数为0（指向对象的最后一个shared_ptr被销毁）时，shared_ptr会销毁这个对象（通过shared_ptr的析构函）

      创建shared_ptr
