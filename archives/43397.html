FireWire火线协议

发表于 2019-12-05 更新于 2019-12-08 分类于 通信 ， 通信协议 阅读次数：

Encoding(编码)

数据传输用两条传输线

  • NRZ coding
  • No stuffing or other measures for clock recovery

STROBE 信号被用于恢复时钟信号

  • 当相同数据被发送时, STROBE 信号会改变它的状
  • Recovery through XOR combination of both signals
  • Advantages:
    • Insensitive for interference
    • Lower bandwidth required

Configuration-of-the-bus

没有专用控制器，每一个总线都能成为root of the tree, Bus assignment is done in three steps:

  • Initialization
  • Tree identification
  • Self identification

Initialization

在总线复位后，每一个subscriber 只知道outgoing connections 的数量

  • One single connection –> Leaf
  • Several connections –> Branch

Tree-identification

  • Method:
    • 每一个leaf发送一个”parent-notify” packet, 标记这个相应的port为”P”(parent)
    • 那个接收到”parent-notify” packet的node会标记对应的port为”ch”(child)
    • 如果在一个node里仅存在一个未被标记的port, 则这个node会使用它发”parent-notify” packet, 并标记”p”
    • 那个等待时间最长，所有ports都标记为ch的将成为root

Self-identification

  • Assigning the node address
    • Root把控制权给#1 port的node
    • 如果这个node有ch, 则继续把控制权传给第一个ch node
    • When a leaf is reached, it will claim the next address that is available and sends a packet with its address and additional status information to the bus
    • After that, control is returned to the parent node that pocesses the next port
    • When a node has processed all of its children, it self-assigns an address and transfers control to its parent node li>

Arbitration

The node sends a request packet to its parent

  • If no other request already reached the parent node, it will forward the request one level up. A data prefix(前缀) packet is send to the other ports in order to suppress(抑制) other request
  • If there is already another request, the last node gets back a data prefix packet to inform the node that the request has been denied

When the request reaches the root, a grant packet is send back to the source node of the request
The requesting node acknowledges with a data prefix and starts the transmission

Fairness

原则上，和root直接相连的那个节点有最大的机会获得总线，为了保证公平，每一个节点只能在一个周期内获得bus一次

  • Therefore every node has an arbitration enable bit, that is cleared after a successful access
  • The node is then not allowed to apply for the bus again

If there was no access to the bus for a certain amount if time(arbitration reset gap, 20$\mu$s), all nodes reset their arbitration enable bit and a new cycle is started.
