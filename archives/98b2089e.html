Valgrind-内存泄漏check

发表于 2020-04-06 更新于 2020-04-07 分类于 WissProg ， C++ 阅读次数：
本文字数： 1.4k 阅读时长 ≈ 1 分钟

简介

Valgrind 是一套类似于 gprof 的动态检测的工具集，由于使用方便，不需修改目标程序源码，输出清晰图文并茂等优势，常被用作后台(特别是linux后台)服务内存泄漏检测、性能打点、竞态检测等。

安装

  • 官方网站 http://valgrind.org/downloads/current.html 下载最新安装包，如 valgrind 3.15.0 (tar.bz2) 解压

  • 常规源码编译安装: ./configure, make, sudo make install

  • 安装完成后, valgrind --version 输出版本号

使用

检测这样一段代码是否有内存泄漏：

  1. start Valgrind with the following command
  • —leak-check : Controls the search for memory leaks when the client program finishes. If set to summary, it says how many leaks occurred. If set to full, each individual leak will be shown in detail
  • —show-leak-kinds : controls the set of leak kinds to show when--leak-check=full is specified. Options are definite, indirect, possible reachable, all and none
  • —track-origins : can be used to see where uninitialized values come from
  • -–verbose: 更详细的信息, 增加错误数统计

  1. cat valgrind-out.txt 读取 log 信息

    在 LEAK SUMMARY 中可以看出丢失了40 bytes, 4 byte*10
