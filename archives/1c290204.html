采样和数据驱动

仿真过程中的竞争问题

在仿真过程中，会产生时序电路中时钟和驱动信号的时许竞争问题，我们需要给出尽量明确的驱动和采样时序。默认情况下，时钟对于组合电路的驱动会添加一个无限最小时间 delta cycle 的延迟，该延迟无法用绝对时间衡量，它要比最小时间单位精度还小。

在仿真器中敲入命令 run 0 可以让仿真器运行一个 delta cycle 的时间。

下面举一个例子来说明：

从仿真图中可以看出，在45ns处，clk1在上升沿采样得到d1的值为1，clk2 采样得到的d1值为2，clk2 和 d1 相比clk1要晚一个 delta-cycle。

如何避免采样的竞争问题

  • 在驱动时，添加相应的人为延迟，模拟真实的延迟行为，同时加大 clk 与变量之间的延迟。
  • 在采样事件前的某段时刻中进行采样，来模拟建立时间的采样要求，确保采样的可靠性。

接口中的clocking

clocking 块基于时钟周期对信号进行驱动或者采样的方式，使得testbench 不再苦恼与如何准确及时地对信号驱动或者采样，消除了信号竞争的问题。

在interface中可以定义多个clocking，同一变量在不同clocking中可以声明不同方向

结论

  • 为了避免可能的采样竞争问题，应该在验证环境的驱动环节就添加固定延迟，使得在仿真波形中更容易体现出时钟与被驱动信号之间的时序前后关系，同时也便于DUT的准确处理和TB的准确采样。
  • 如果TB在采样从DUT送出的数据，在时钟与被驱动信号之间在存在delta-cycle时，应该考虑时钟采样沿的更早时间段去模拟建立时间要求采样，这种方法也可以避免由于delta-cycle问题带来的采样竞争问题。
  • 当我们把clocking运用到interface中，用来声明各个接口与时钟的采样和驱动关系后，可以大大提高数据驱动和采样的准确性，从根本上消除采样竞争的可能性。
