数据类型

内建数据类型

logic 类型

Verilog 有两种基本数据类型：reg 和 wire, 有时候初学者很难分清两者的区别。SystemVerilog 对 reg 进行了改进，使得它除了作为一个变量以外，还可以被连续赋值，门单元和模块所驱动，这种数据类型被称为 logic。

注意：

由于logic 类型只能有一个驱动，所以可以用它来查找网单中的漏洞。把所有的信号都声明为 logic 而不是 reg 或 wire，如果存在多个驱动，那们编译时就会出现错误。如果希望有多个驱动，这些信号就需要被定义成线网类型，如 wire。

二值逻辑/四值逻辑

与 logic 相对应的是 bit 类型，它们均可构建 vector，区别在于：

  • logic 为四值逻辑，即可以表示 0，1，x，z
  • bit 为二值逻辑，只可以表示 0，1

那为什么SV要在已经有了四值逻辑的基础上还要再引入二值逻辑呢？

这是因为SV 希望将硬件和软件分离开，硬件使用四值逻辑，而软件使用二值逻辑。二值逻辑类型有利于提高仿真器的性能并减少内存的使用量。

四值逻辑： integer，logic，reg，net-type(wire, tri)

二值逻辑： byte，shortint，int，longint，bit

有符号类型： byte，shortint，int，longint，integer

无符号类型： bit，logic，reg，net-type(wire, tri)

存储空间考量

很大SV 仿真器在存放数组元素时使用32 bit 的字边界，所以byte, shortint, int 都是放在一个word中，而longint则放到两个word中

bit [3][7:0] b_pack; // 合并数组

bit [7:0] b_unpack[3] // 非合并数组

从上图可得b_unpack实际会占据3个word的存储空间，但b_pack则之会占据一个word空间

如果使用logic类型来存储，即logic [3][7:0] l_pack;, logic [7:0] l_unpack[3]; 那么它们所占据的实际存储空间是多少呢？

一位logic有四种状态，2^2=4，所以需要两位。
l_pack: 24 * 2 = 42，需两个word

l_unpack: 3(8\2) ，需三个word

动态数组

前面介绍的定宽数组，其宽度在编译时就确定了。但如果直到程序执行之前都还不知道数组的宽度，我们就需要用到动态数组，它可以在仿真时分配空间或调整宽度

队列

队列结合了链表和数组的优点，可以在任何地方添加或删除元素，并且通过索引实现对任一元素的访问。

队列声明使用[$]，队列标号从0到$

队列中的元素是连续存放的，属于合并数组，因此赋值不需要‘

关联数组

如果你只是偶尔需要创建一个大容量的数组，那么动态数组已经足够好用了，但是如果需要超大容量的呢？动态数组的限制在于其存储空间在一开始创建时即被固定下来，那么对于超大容量的数组，这种方式无疑存在着浪费，因为很有肯该大容量的数组中有相当多的数据不会被存储和访问。

关联数组可以用来保存稀疏矩阵的元素。这意味着当你对一个非常大的地址空间进行寻址时，SV 只为实际写入的元素分配空间。

结构体

Verilog 最大的缺陷之一是没有数据结构，在SV中可以使用struct语句创建结构，和C类似。struct 只是一个数据的集合，所以它是可综合的

枚举类型

规范的操作码和指令有利于代码的编写和维护，它比直接使用’h01这样的常量使用起来的可读性和可维护性更好。

枚举类型enum经常和typedef搭配使用，便于用户自定义枚举类型，共享使用。

如果没有特别指出，枚举类型会被当成int类型储存，nstate是一个int类型，右侧是枚举类型可以直接赋值给左侧的整型 INT = ENUM; 若右侧是整型，左侧是枚举类型，那右侧需要做类型转换，ENUM = T’(INT)

字符串

SV中string类型可以用来保存长度可变的字符串，单个字符是byte类型。与C语言不同的是，字符串的结尾不带标识符null
