A model is an abstract representation of some system that is easier to understand in a given context than the actual system.

Introduction to Modeling

Physical Models

  • A physical objekt that mimics some properties of a real system
  • Through prototyping process
    • Protyping is the process of quickly putting together a working model in order to test various aspects of a design, illustrate ideas or features and gather early user feedback.

Mathematical Models

A description of a system where the relationship between variables of the system are expressed in a mathematical form, e.g. Ohm’s Law

  • Deterministic models
    the input and output variables are not subject to random fluctuations(随机波动), so that the system is at any time entirely defined by the initial conditions chosen
  • Stochastic models
    at least one of the input or output variables is probabilistic or involves randomness

Modeling VS Programming Languages

Modeling on System Level

Similution: A simulation of a system is the operation of a model of the system, as an imitation(模仿) of the real system.

Applied to complex systems that are impossible to solve mathematically.

Reasons for Simulation:

  • Complexity: Developing comples systems is possible
  • Economy
  • Time scale of the dynamics of the system is not compatible with that of the observer
    e.g. It takes millions of years to observe small changes in the development of the universe, whereas changes can be quickly simulated in a computer
  • Suppression of disturbances/accuracy
  • Accessibility: Nodes in an ECU, for example, can be monitored easily, whereas they are no longer accessible after realization.
  • Flexibility: Ability to quickly implement changes

Goals of simulation

  • determine the correct functioning of a dsign with the help of the simulation
  • Simulation programs have to recognize, if and where errors occur(not filled the desired function, delay, tolerance)
  • However, Physical constraints are included seldomly(很少) into the simulation: logic simulation, but voltage and current are not always in expected time and quality

Some application areas:

  • manufacturing systems
  • mitary weapons
  • communications networks

Advantages:

  • When mathematical analysis methods are not avaliable, simulation may be the only investigation tool
  • Simulation may provide a simpler solution
  • Allow time compression(压缩) or expansion
  • Allows comparisons of alternative designs or alternative operating policies

Disadvantages:

  • Time consuming
  • An invalid model may result with confidence in wrong results.

Timing: How do we advance time?

Time handling

  • Time-slicing: move forward in equal time intervals(synchronous)
  • Discrete-event simulation(DES)
    • Time moves from event to event
    • periods of inactivity are skipped over by jumping the clock form event time to event time
  • Continuous simulation
    • refers to a computer model of a physical system
    • continuously tracks system response
    • according to a set of equations typically involving differential equations
  • DES is asynchronous
    • time hop(跳跃)
    • Because DES don’t have to simulate every time slice, faster than continuous simulation
    • slices are much larger than in continuous simulations

Finite State Machines

A FSM is a mathematical model of computation used to design computer preograms, embedded systrms and sequential logic circuits.

The machine is only one state at a time.

Mealy machine: outputs are a function of both inputs and current state
Moore machine: outputs are a function of current state only

Statecharts

A visual formalism for specifying hierarchical state machines, especially useful for modeling reactive systems (systems which continuously react to internal or external events without terminating)

History state
A composite state (复合状态) may contain shallow history and deep history states.

  • Shallow history state is a reference to the most recently visited state on the same hierarchy level within the composite state.(同级状态)
  • Deep history state is a reference to the most recently visited simple state within the composite state.(包含子状态)

Petri-nets

Petri-net = specific directed, bipartite, marked graph. Two sets of nodes are Places and Transitions.

w(p,t): the weight of the edge from p to t.

Transition Rules

  • A transition is called enable if each of its input places p is marked with at least w(p,t) tokens.
  • An enabled transition switches by removing w(p,t) tokens from each input places p and putting w(t,p) tokens to each output place
  • An enabled transition can switch but it does not have to $\rarr $ not deterministic

A transition is called enabled (firable) if each of its input places contains at least one token.某个transition 的所有输入位置中都必须有marking, 并且当输入位置有多根弧线指向这个transition时, 该输入的marking数$\geq$ 弧线数

Marking 移动规则

  • 从发生transition t 的所有输入Place 中取出marking, 每个Place取出的标记数等于该位置指向t 的弧线数
  • 然后将marking送入t的所有输出位置, 送入每个位置的marking等于t指向该位置的弧线数

considering “weights”

Behavioral characteristics

  • Reachability: Starting from a given marking, you examine whether a specific new marking could be reached.
    A marking $M_n$ is called reachable by marking $M_0$, if there is a sequence


    of such transitions, that each $t_{i+1}$ is $M_i$ enabled and switches from $M_i$ to $M_{i+1}$.

    • The set of all markings reachable from M is described as R(M)
    • Problem of reachability:
      • given: initial marking $M_0$, target marking M
      • question: Is M reachable from $M_0$, i.e. is $M \in R(M_0)$ ?

  • Safeness: In a network with an initial marking you check whether new markings lead to an excess(超过) of a specific capacity of one or more places. A network is called k-safe, if capacity k is not being exceeded for all places of a network.

  • Liveliness/Deadlock: A transition t is called alive if it can be made enabled from any reachable marking. A petri net is called alive if all transitions are alive, if for each transition t a marking M’, in which t is enabled, is reachable from the position of each marking $M \in R(M_0)$.

  • Conservation: A Petri net is conservative if the number of tokens in the net is conserved(输入等于输出)

  • Reversibility: A Petri net is called reversible, if for all $M \in R(M_0)$ the initial marking $M_0$ can be reached from the position of M(从M回到$M_0$)
    Remark: Existence of the reversibility ensures that the system can always recover the normal behavior and is important for systems subject to failures.

(Message) Sequence charts

Explicitly indicate exchange of information, one dimension(usually vertical dimension) reflects time, the other reflects distribution in space.

Object Life Spans

  • Creation: create message and object life starts at that point.
  • Activation: Place on the lifeline where object is activated
  • Deletion: Placing an ‘X’ on lifeline and object’s life ends at that point

Object interaction

  • Self-call: a message that an Object sends to itself
  • Condition: indicates when a message is sent. The message is sent only if the condition is true

MSC textual representation

Pros:

  • Appropriate for visualizing schedules, e.g. in transportation
  • Standard defined, semantics also defined

Cons:

  • describes just one case, no timing tolerances
  • Only for small communication examples with no more than 1-3 messages synchronously

Use Case Diagram

A Use Case is the specification of a set of actions performed by a system, which yields an observable result that is of value for one or more actors of the system.

Element:

  • Actors: Roles that people can fill when they interact with the system
  • Use Case: A use case represents a coherent (一致) unit of functionality provided by a system. It describes a whole class of interactions
